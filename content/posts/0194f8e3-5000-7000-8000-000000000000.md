React나 Vue를 써봤다면 한 번쯤 이런 생각 해보셨을 겁니다. _"왜 이렇게 번들이 무겁지?"_ 스벨트(Svelte)는 바로 그 지점에서 출발합니다. React, Vue와 함께 현대 웹 프런트엔드를 이끄는 주요 기술이지만, 동작 방식이 근본적으로 다릅니다.

가장 쉽게 설명하면 **"언제 동작하느냐"** 의 차이입니다.

React와 Vue는 배포 시 해당 라이브러리 자체가 번들에 포함됩니다. 그래서 브라우저에서 실행되는 내내 프레임워크가 함께 동작하며, 화면이 바뀔 때마다 가상 DOM을 통해 변경된 부분을 계산합니다. 반면 스벨트는 배포 전 컴파일 시점에 이미 그 작업을 끝내버리기 때문에, 브라우저에는 순수 자바스크립트만 전달됩니다. 프레임워크가 런타임에 껴있지 않으니 그만큼 가볍고 빠릅니다.

덕분에 번들 크기가 작고, 브라우저 자원을 훨씬 효율적으로 씁니다. 
특히 성능에 민감한 환경이나 가벼운 프로젝트라면 스벨트 도입이 괜찮습니다.

다만 현실적인 한계도 있습니다. 아직 생태계와 서드파티 라이브러리가 React에 비해 부족한 편이라, 실무에서 도입할 때는 이 부분을 꼭 고려해야 합니다.

## 개발 환경 설정 가이드
Svelte는 오직 코드에만 집중할 수 있도록 매우 간결한 시작 방법을 제공합니다. 웹 브라우저에서 즉시 실습하거나, 로컬 PC에 안정적인 작업 공간을 마련하는 두 가지 방법을 소개해 드립니다.

### 웹 브라우저에서 즉시 진행(Playground)
**Svelte**는 별도의 로컬 설정 없이도 웹에서 즉시 코드를 작성하고 결과를 확인할 수 있는 실습 환경(REPL)을 제공합니다. 해당 환경을 통해 스벨트의 문법을 직접 구현해보고, 컴파일러가 생성하는 결과물을 실시간으로 확인해 보실 수 있습니다.
- **URL:** https://svelte.dev/playground


### 로컬 개발 환경 구축하기 (Mac 기준)
터미널을 통해 몇 가지 명령어만으로 Svelte 프로젝트를 즉시 시작할 수 있습니다.

1. **사전 준비** : 먼저 시스템에 **Node.js**와 **npm**이 설치되어 있는지 확인이 필요합니다. 터미널을 열고 아래 명령어를 입력하여 버전을 체크해 주세요
```
node -v
npm -v
```

2. **프로젝트 생성**: Svelte 프로젝트를 생성하기 위해 가장 널리 쓰이는 빌드 도구인 **Vite**를 활용합니다. 터미널에 다음 명령어를 입력해 주세요.
```bash
npm create vite@latest my-svelte-app
```

> **설정 팁:** 명령어를 입력하면 프레임워크 선택 화면이 나타납니다. `Svelte`를 선택하신 후, 선호에 따라 `JavaScript` 또는 `TypeScript`를 선택해 주시면 됩니다.


3. **의존성 설치 및 실행**:  생성된 프로젝트 폴더로 이동하여 필요한 패키지들을 설치하고 개발 서버를 가동합니다.
```bash
# 프로젝트 폴더로 이동 
cd my-svelte-app 

# 필요한 패키지 설치 
npm install 

# 개발 서버 실행 
npm run dev
```
모든 과정이 완료되었다면 브라우저 주소창에 `http://localhost:5173`을 입력해 보세요.
